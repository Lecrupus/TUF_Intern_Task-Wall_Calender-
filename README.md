# Basketball Anthology Calendar (2026 Edition)

A professional-grade, interactive React calendar application designed with a high-fidelity skeuomorphic interface. This project serves as a "Strategist's Notebook," blending the visual appeal of physical sports memorabilia with advanced digital features including 3D animations, synchronized audio, and persistent data management.

---

## Feature Deep-Dive: Advanced Note System

The application implements a non-linear note-taking system that allows users to attach strategic data to specific temporal markers.

### 1. Dynamic Note Dictionary
Instead of a simple array, the notes are stored in a **Key-Value Dictionary Object**. This ensures O(1) lookup time when rendering the calendar grid.
* **Key Generation:** Every note is tied to a unique string key generated by the `getNoteKey` function. 
* **Storage Pattern:** The system uses `JSON.stringify` to persist the entire dictionary to `localStorage`. Upon component mounting, the `useEffect` hook hydrates the state from the browser's disk, ensuring data persistence across sessions.

### 2. Dual-Pointer Date Range Selection
The calendar supports a sophisticated range-selection feature that goes beyond selecting a single day. This is managed through two primary states: `selectionStart` and `selectionEnd`.

* **Logic Flow:**
    1.  **First Click:** Sets the `selectionStart`. The UI highlights this single cell.
    2.  **Second Click (Future Date):** Sets the `selectionEnd`. The application logic automatically calculates the interval between these two pointers.
    3.  **Visual Feedback:** The grid uses a conditional CSS logic to apply the `bg-sky-500` class to all days where `(dayObj.date >= selectionStart && dayObj.date <= selectionEnd)`.
* **Range-Based Notes:** When a range is selected, the application generates a composite key (e.g., `2026-04-10_to_2026-04-15`). This allows "Campaign" or "Tournament" notes to be stored separately from individual daily logs.

## Technical Stack

* **React.js:** Utilized for declarative UI rendering and efficient reconciliation of the calendar grid.
* **Tailwind CSS:** Employed for the entire design system, including custom utility classes for 3D perspective and dynamic theming.
* **Web Audio API:** Managed through `useRef` to maintain a persistent audio context for the page-flip sound effects.
* **JavaScript Date Object:** Used for calculating dynamic month lengths, leap years, and starting day offsets.

## Detailed Component Architecture

### 3D Perspective Engine
The core of the interface is built on a 3D CSS perspective engine. By applying `perspective: 1200px` to the parent container, child elements gain a Z-axis depth. When the month changes, the `animate-page-flip` class triggers a rotation around the X-axis, anchored at the top (`origin-top`), simulating the movement of a real spiral-bound desk calendar.

### Indian Gazetted Holiday Engine
The application contains an internal database of 2026 Indian holidays.
* **Implementation:** During the `.map()` execution of the calendar days, each date is formatted into a `YYYY-MM-DD` string and cross-referenced against a constant holiday object. 
* **UI Integration:** Detected holidays receive a specific `bg-orange-500` marker and a hover-triggered tooltip.
